재현 조건이 단순한 버그가 제일 무섭다. 이 버그는 "로그아웃 버튼을 누른다"가 전부였다.

기대 동작은 당연히 인증 상태 초기화 후 로그인 화면으로 이동. 그런데 종종 401 에러가 뜨면서 화면이 하얗게 됐다. "종종"이라는 게 더 골치였다. 항상 재현되지 않았다.

처음엔 로그아웃 API 호출 타이밍 문제라고 생각했다. 토큰을 삭제하기 전에 어딘가에서 인증이 필요한 API를 먼저 호출하는 경쟁 조건처럼 보였다. 로그아웃 플로우 코드를 한참 들여다봤는데 원인이 거기 없었다.

## Profiler를 켰다

React DevTools Profiler를 열고 로그아웃 버튼을 눌렀다. 예상하지 못한 걸 봤다.

**Header 전체가 리렌더링됐다.** 로그아웃 처리 중에.

Header에 `useModal`이 있었다. 이 훅은 `ModalContext`를 구독하고 있었다.

```tsx
const ModalContext = createContext<ModalContextType>({
  modals: [],
  open: ...,
  close: ...
});

export const useModal = () => useContext(ModalContext);

// Header 어딘가
const { modals } = useModal(); // modals 배열 전체 구독
```

Context의 문제는 **구독 단위를 제어할 수 없다**는 것이다. `ModalContext`에는 `modals` 배열과 `open`, `close` 함수가 함께 들어 있었다. Context 값이 조금이라도 바뀌면 `useContext`를 쓰는 컴포넌트가 전부 리렌더링된다. Header는 모달 함수만 쓰면 됐는데, `modals` 배열이 바뀔 때마다 같이 렌더링됐다.

로그아웃 플로우 중에 모달 상태가 변경됐고, 그로 인해 Header가 리렌더링됐고, 그 안에서 인증 확인 로직이 예상치 못한 타이밍에 실행됐다. 401이 뜬 이유가 여기 있었다.

## Context API의 한계

`ModalProvider` 외에도 `RoutePathProvider`, `AuthProvider` 같은 Context 기반 Provider들이 있었다. 확인해보니 모두 비슷한 구조적 문제를 갖고 있었다.

Context를 전역 상태 관리 도구로 쓰는 건 처음부터 용도에 맞지 않는다. React 공식 문서도 Context를 prop drilling을 피하기 위한 도구로 설명하지, 상태 관리 도구라고 설명하지 않는다. 자주 바뀌는 상태, 여러 컴포넌트에서 부분적으로 구독하는 상태는 처음부터 Zustand 같은 도구로 다뤄야 했다.

## Zustand로 교체

`useModalStore`로 바꾸면서 selector로 필요한 것만 구독하도록 했다.

```ts
const useModalStore = create<ModalState>()(
  immer((set, get) => ({
    modals: [],
    handleModalOpen: (modal) => (e) => {
      e?.stopPropagation();
      get().handleModalAdd(modal);
    },
    handleModalAdd: (modal) =>
      set((state) => {
        state.modals.push(modal);
        document.body.style.cssText = "overflow: hidden";
      }),
    handleModalClose: () =>
      set((state) => {
        state.modals.pop();
        if (state.modals.length === 0) {
          document.body.style.cssText = "overflow: auto";
        }
      }),
  }))
);
```

Header에서는 함수만 구독한다. `modals` 배열이 바뀌어도 Header는 렌더링되지 않는다.

```tsx
// Header
const handleModalOpen = useModalStore((state) => state.handleModalOpen);

// 모달 렌더링 영역에서만 배열 구독
const modals = useModalStore((state) => state.modals);
```

`RoutePathProvider`, `AuthProvider`도 동일하게 전환했다. 이 변경들을 `packages/stores`에 공용 패키지로 이동하니까 10개 앱에 일괄 적용됐다.

---

Profiler를 다시 열었을 때 Header가 더 이상 리렌더링되지 않는 걸 확인했다. 버그도 재현되지 않았다.

버그 원인을 찾는 데 생각보다 오래 걸렸다. 처음에 로그아웃 플로우만 들여다보다가 시간을 썼는데, Profiler를 먼저 켰으면 더 빨리 찾았을 거다. 이후로는 렌더링 관련 버그 의심이 생기면 코드 먼저 보기 전에 Profiler를 먼저 켠다.
