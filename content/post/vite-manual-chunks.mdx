초기 번들 사이즈가 4,990kB였다. 사용자가 앱을 처음 열 때 거의 5MB짜리 JavaScript를 받아야 했다. 어드민 앱이라 일반 사용자보다 기기가 좋긴 하지만, 그래도 너무 컸다.

처음에는 lazy import만 적용하면 될 거라고 생각했다. 무거운 컴포넌트 몇 개를 `lazy()`로 감싸면 초기 번들에서 빠질 테니까. 실제로 해봤는데 기대보다 효과가 작았다. 문제가 다른 곳에 있었다.

## 배럴 파일이 범인이었다

모노레포에서 공용 패키지를 만들면 자연스럽게 이런 구조가 생긴다.

```ts
// packages/components/index.ts
export { Button } from "./Button";
export { Input } from "./Input";
export { Table } from "./Table";
// ... 206개
```

앱에서는 `import { Button } from "@repo/components"` 한 줄로 편하게 가져다 쓴다. 이게 배럴 파일이다.

편리한데 번들러 입장에서는 최악이다. `Button` 하나를 가져오려고 해도 Vite는 `index.ts` 전체를 분석해야 한다. Tree-shaking이 작동하긴 하지만, 모노레포에서 여러 패키지의 배럴 파일이 중첩되면 분석해야 할 모듈 그래프가 기하급수적으로 커진다.

이게 개발 서버에도 영향을 줬다. 첫 기동에 53초가 걸렸다. 팀에서 한동안 그냥 쓰고 있었는데 돌이켜보면 매일 1분씩 낭비하고 있었던 거다. VSCode 자동완성도 10초씩 멈췄다. TypeScript 언어 서버가 배럴 파일을 타고 수천 개의 타입을 추론하느라 멈추는 거였다.

배럴 파일 1,600개를 제거하고 직접 경로로 임포트하도록 바꿨다.

```ts
// Before
import { Button } from "@repo/components";

// After
import { Button } from "@repo/components/Button";
```

ESLint 규칙을 추가해서 새로 배럴 파일이 생기는 걸 막았다. 이것만으로 개발 서버 기동이 **53초 → 6초**, 자동완성이 **10초 → 2초**가 됐다.

## lazy import는 그래도 필요했다

배럴 파일을 제거하니까 번들 사이즈가 많이 줄었는데, PDF 렌더러나 지도 컴포넌트처럼 라우트 하나에서만 쓰는 무거운 라이브러리는 여전히 초기 번들에 포함돼 있었다.

```tsx
// 처음 접속할 때 이걸 왜 받아야 하나
import { SunEditor } from "suneditor-react";
import { PDFViewer } from "@react-pdf/renderer";
```

이건 lazy import로 해결했다. 해당 라우트에 진입할 때 처음 로드된다.

```tsx
const SunEditor = lazy(() => import("suneditor-react"));
const PDFViewer = lazy(() =>
  import("@react-pdf/renderer").then((m) => ({ default: m.PDFViewer }))
);
```

## manualChunks는 캐싱을 위한 것

번들 크기가 많이 줄었는데, 한 가지 더 신경 쓰이는 게 있었다. Vite 기본 설정은 `node_modules`를 하나의 `vendor` 청크로 묶는다. `dayjs` 하나만 업데이트해도 `vendor` 전체가 캐시 무효화된다. 즉 사용자가 매 배포마다 수백 kB를 다시 받아야 한다.

`manualChunks`로 라이브러리를 각자 청크로 분리했다.

```ts
export const createManualChunks = (id: string): string | undefined => {
  if (id.includes("node_modules")) {
    return generateChunkName({
      list: CHUNK_LIST.library,
      prefix: CHUNK_PREFIX.library,
      id,
    });
  }
  // ...
};
```

```ts
export const CHUNK_LIST = {
  library: [
    "@tanstack/react-query",
    "react-dom",
    "react-router-dom",
    "zustand",
    "lodash-es",
    "dayjs",
    "axios",
    // ...
  ],
};
```

결과는 `__Libraries-react-dom.[hash].js`, `__Libraries-dayjs.[hash].js` 같은 청크들이다. `dayjs`를 업데이트해도 `react-dom` 청크는 캐시 그대로 유지된다.

이 설정을 `packages/utils/vite.ts`에 공용 함수로 두고 10개 앱이 모두 가져다 쓴다. 청크 전략을 한 곳에서 관리하면 된다.
