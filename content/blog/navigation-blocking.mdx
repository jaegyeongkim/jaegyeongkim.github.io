폼이 있는 페이지에서 뒤로가기를 누르면 입력한 내용이 사라진다. 어드민 앱에서는 이게 꽤 큰 문제다. 긴 폼을 절반쯤 작성하다가 실수로 네비게이션을 누르면 처음부터 다시 해야 한다.

"저장하지 않고 나가시겠습니까?" 확인 모달이 필요하다.

브라우저 기본 동작으로 `beforeunload` 이벤트가 있긴 한데, 이건 탭을 닫거나 주소창에서 이동할 때만 동작한다. SPA 내부 라우팅 — React Router로 페이지 전환 — 에는 반응하지 않는다. 두 가지를 모두 처리해야 했다.

## React Router의 useBlocker

React Router v6.7부터 `useBlocker` 훅이 공식 지원된다. 내부 라우팅이 발생하기 전에 가로채서 처리할 수 있다.

```ts
const useNavigationBlocking = (
  isDirty: boolean,
  options?: UseNavigationBlockingOptions,
) => {
  const [destinationLocation, setDestinationLocation] = useState(location);
  const [isConfirmedNavigation, setIsConfirmedNavigation] = useState(false);

  const handleNavigationBlock = useCallback(
    ({ currentLocation, nextLocation }): boolean => {
      if (!isDirty) return false;
      if (isConfirmedNavigation) return false;

      const currentPath = `${currentLocation.pathname}${currentLocation.search}`;
      const nextPath = `${nextLocation.pathname}${nextLocation.search}`;

      if (currentPath !== nextPath) {
        setDestinationLocation(nextLocation);
        handleNavigationBlockingModalOpen(handleConfirmNavigation)();
        return true; // 이동 차단
      }

      return false;
    },
    [isConfirmedNavigation, isDirty, handleNavigationBlockingModalOpen],
  );

  useBlocker(enabled && isDirty ? handleNavigationBlock : false);
};
```

`handleNavigationBlock`이 `true`를 반환하면 라우팅이 차단된다. 이때 목적지(`destinationLocation`)를 state에 저장해두고 확인 모달을 연다.

모달에서 "확인"을 누르면 `isConfirmedNavigation`이 `true`가 되고, 그 시점에 저장해뒀던 목적지로 직접 이동한다.

```ts
useEffect(() => {
  if (!isConfirmedNavigation) return;
  navigate(`${destinationLocation.pathname}${destinationLocation.search}`);
  resetConfirmedNavigation();
}, [isConfirmedNavigation, destinationLocation]);
```

## beforeunload는 별도로

브라우저 탭 닫기, 새로고침, 주소창 직접 입력은 React Router가 관여하지 못한다. 이건 `beforeunload` 이벤트로 처리한다.

```ts
const handleBeforeUnload = (event: Event): void => {
  event.preventDefault();
  event.returnValue = false;
};

useEffect(() => {
  if (!enabled) return;

  if (isDirty) {
    window.addEventListener("beforeunload", handleBeforeUnload);
  }

  return () => {
    window.removeEventListener("beforeunload", handleBeforeUnload);
  };
}, [enabled, isDirty]);
```

`isDirty`일 때만 이벤트 리스너를 붙이고, 폼 상태가 바뀌거나 컴포넌트가 언마운트될 때 제거한다. 클린업을 빠뜨리면 다른 페이지에서도 브라우저 이탈 경고가 뜨는 문제가 생긴다.

## useFormWithNavigationBlocking으로 마무리

`useNavigationBlocking`을 직접 쓰려면 `formState.isDirty`를 꺼내서 넘겨야 한다. 이걸 묶어서 `useFormWithNavigationBlocking`을 만들었다.

```ts
const useFormWithNavigationBlocking = <T extends FieldValues>(
  options: UseFormWithNavigationBlockingOptions<T>,
): UseFormReturn<T> => {
  const { hasNavigationBlocking, ...formOptions } = options;
  const formMethod = useForm<T>(formOptions);

  useNavigationBlocking(formMethod.formState.isDirty, {
    enabled: hasNavigationBlocking,
  });

  return formMethod;
};
```

`useForm`과 완전히 같은 인터페이스다. `hasNavigationBlocking: true`만 추가하면 자동으로 붙는다.

```ts
// 폼 있는 페이지에서
const formMethod = useFormWithNavigationBlocking({
  hasNavigationBlocking: true,
  defaultValues: { ... },
});
```

`hasNavigationBlocking: false`로 두면 차단 없이 동작하기 때문에, 읽기 전용 페이지나 간단한 폼에서는 그냥 끌 수 있다.

---

한 가지 주의할 게 있다. `isConfirmedNavigation` 상태 관리가 미묘하다. "확인"을 누른 뒤 navigate하고 나서 `resetConfirmedNavigation()`을 호출해 다시 `false`로 돌려야 한다. 이걸 빠뜨리면 다음 페이지에서 폼을 작성할 때 차단이 작동하지 않는다. 처음 구현할 때 이 부분에서 한 번 걸렸다.
