`content/post/AGENTS.md`에 정의된 글쓰기 가이드를 먼저 읽고, 그 구조/오프닝/문체/체크리스트를 그대로 따라 새 글을 작성한다.

주제: $ARGUMENTS (비어 있으면 사용자에게 어떤 문제/작업을 다룰지 먼저 물어본다)

> ⚠️ 이 가이드 이전 문체(반말 회고체)로 쓰인 기존 글들은 참고용으로 읽지 않는다 — `content/post/AGENTS.md`에 명시된 존댓말 + 8단계 구조만 따른다. 기존 글의 내용도 수정하지 않는다.

## 진행 순서

1. `content/post/AGENTS.md`를 읽는다.
2. 주제에 대해 아래 정보를 사용자에게 확인하거나, 대화 맥락에서 파악한다. 빠진 정보(특히 결과 수치)는 지어내지 말고 사용자에게 묻는다.
   - 오프닝에 쓸 구체적 장면/상황 (에러, 현장 반응, 성장 배경 등)
   - 배경 / 왜 이 문제를 풀어야 했는지, 어떤 원칙으로 풀기로 했는지
   - 검토했던 대안들 (기술/방향 비교표에 쓸 내용)
   - 사이클별로: 이번에 부딪힌 어려움, 어떻게 해결했는지, 예상 못한 문제가 있었다면 다음 사이클로
   - 종합 결과 수치 (Before/After)
   - 배운 점 / 교훈
   - 남은 이슈 (있다면)
3. `content/post/{slug}.mdx`를 작성한다. `content/post/AGENTS.md`의 "글의 구조" 순서(오프닝 → 배경(문제→결정→비교와 선택) → N차 문제 반복 → 종합 결과 → 회고 → 남은 이슈)를 따르고, 존댓말·bold 강조 리스트·💡 콜아웃·표를 가이드대로 사용한다.
4. `lib/posts.ts`의 `POSTS` 배열에 새 글의 slug, title, description, date, tags를 추가한다.
5. 작성한 글이 가이드의 체크리스트를 모두 만족하는지 스스로 검토한 뒤 결과를 요약해서 보고한다.
